# 📊 マーケティング・ダッシュボード（広告代理店/チーム共用版） 取扱説明書

## 🌟 システム概要
本ツールは、広告代理店のチーム全員が共通の「顧客リスト」を参照し、各クライアントのWEB広告効果とGA4データを自動統合して分析するためのプロフェッショナル向けツールです。

---

## � 公開と共有の手順（GitHub Pages）

本ツールを社員全員で使うために、URLを発行する手順です。

1. **GitHubリポジトリを作成**
   - 新しいリポジトリ（Private推奨）を作成し、本フォルダのファイルをすべてアップロード（Push）します。
2. **Pagesを有効化**
   - リポジトリの `Settings` > `Pages` を開き、`Build and deployment` > `Branch` で `main` を選択して `Save`。
3. **URLの確認**
   - 数分後に `https://[ユーザー名].github.io/[リポジトリ名]/` というURLが発行されます。

---

## � セキュリティ設定（重要）

外部の人間がURLにアクセスしてもデータを閲覧できないよう、ログイン制限をかけます。

### 1. ドメイン制限の設定
`app.js` の冒頭にある `ALLOWED_DOMAINS` は現在以下のように設定されています。
```javascript
// 貴社アカウント（@ico-ad.co.jp）のみログイン可能
const ALLOWED_DOMAINS = ['ico-ad.co.jp']; 
```
これにより、部外者がGoogleアカウントを持っていても、ログイン時にエラーが表示されアクセスが拒否されます。

### 2. Google Cloudの設定更新
GitHub PagesのURLが確定したら、Google Cloud Consoleの「承認済みの JavaScript 生成元」にそのURLを追加する必要があります。
- 追加先: `https://[ユーザー名].github.io`

## 👥 チームでの顧客・データ管理

### 1. マスター顧客名簿の同期
全社員で同じ顧客リストを表示させるには、以下のスプレッドシートを更新します。
[マスター名簿スプレッドシート](https://docs.google.com/spreadsheets/d/1Zg9kByoLMidbe5i4FEGzgQPGExHTZYEeh6WouGaSINA/edit?usp=sharing)
- **A列**: 顧客名
- **B列**: サイト名
- **C列**: スプレッドシートID（またはURL）
※ここに追記するだけで、全社員のダッシュボードに自動で顧客が追加されます。

### 2. 各クライアント用スプレッドシート
各顧客のスプレッドシートは、以下の3タブ構成で運用してください。
- `GA4`: セッション、成約数などの成果データ
- `広告費`: 媒体ごとのコストデータ
- `施策ログ`: 実施した施策の内容と日付

---

## � トラブルシューティング
- **ログインボタンが表示されない**: URLがGoogle Cloudの「承認済みの JavaScript 生成元」に正しく登録されているか確認してください。
- **データが読み込まれない**: 各スプレッドシートの共有設定が「リンクを知っている全員（閲覧者）」または「社員全員」になっているか確認してください。
